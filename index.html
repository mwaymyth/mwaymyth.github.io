    Introduction

    Welcome to the Phood API! You can use our API to access Phood API endpoints, which provides access to users, payment types, payment balance and other phood delivery resources.

    You can access the Phood API GraphQL endpoing using GraphiQL, curl or any HTTP client. You SHOULD use the POST method for all requests. All responses come in standard JSON.

    Here is how you can use GraphiQL to acccess the Phood API.

    This example API documentation page was created with Slate. Feel free to edit it and use it as a base for your own API's documentation.

    Authentication

    To authorize, use this code:

      # put Authorization in the *Header Name* and Bearer userToken as the **Header Value*


    # With shell, you can just pass the correct header with each request
curl \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer userToken" \
  -X POST https://phood-api-prod.herokuapp.com/graphql

    Make sure to replace userToken with your access token.

    The Phood API uses OAuth2 for authentication.

    OAuth2 requests must be authenticated with a valid access token passed as bearer token (userToken). To use the bearer token, construct a normal HTTPS request and include an Authorization header with the value of Bearer

    Phood expects for the bearer token to be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer userToken

    You must replace userToken with your personal API key.

    GraphQL Endpoint

    https://phood-api-prod.herokuapp.com/graphql

    Login

    To login, use this code:

    curl \
  -H "Content-Type: application/json" \
  -X POST https://phood-api-prod.herokuapp.com/graphql \
  -d '{ "query": " mutation Authenticate($userEmail:String!, $userPassword:String!){
      authenticate(email:$userEmail, password:$userPassword) }",
      "variables": { "userEmail": "youremail@gmail.com", "userPassword": "password1234"} }'

    mutation {
    authenticate(email:"email@gmail.com", password:"password")
}

    Make sure to replace youremai@gmail.com with a vaild email and password1234 with a valid password.

    The above command returns JSON structure like this:

    {"data":
  {
    "authenticate":"userToken"
  }
}

    In order to generate a bearer token, you must have a valid Phood account.

    Login will return an access token that will be used to authenticate the requests.

    Query Parameters

    Parameter Description
    email string A valid email associated with a Phood account
    password string A vaild password associated with a Phood account

    Interacting with the API

    Request Requirements

    The content type can go above or below the authenticate header but MUST be included.

    curl \
  -H "Content-Type: application/json"

    The Content-Type Header

    The content type of a query MUST be application/json

    Content-Type: application/json

    HTTP Request

    POST https://phood-api-prod.herokuapp.com/graphql

    curl \
  -H "Content-Type: application/json" 
  -H "Authorization: Bearer userToken" \
  -X POST https://phood-api-prod.herokuapp.com/graphql \
  -d '{ "query": "GraphQL script with reference vars $var1, $var2",
        "variables": {
          "var1" : "Hello",
          "var2" : 10
        }
      }

    query {
  GraphQL script with reference vars $varOne, $varTwo
}

    # Query Variables
{
  "varOne" : "Hello",
  "varTwo" : 10
}

    The Message Body

    The message body MUST be a JSON object. It MUST contain a query key and MAY contain a variables key.

    The value of query MUST be a single JSON string tha contains a valid GraphQL script(query or mutation).

    If you reference variables in your GraphQL script, the value of variables MUST be a JSON object containg the key/vaulue pairs for each variable.

    curl "http://example.com/api/kittens"
  -H "Authorization: meowmeowmeow"

    Query Parameters

    Parameter Default Description
    include_cats false If set to true, the result will also include cats.
    available true If set to false, the result will include kittens that have already been adopted.
    Remember — a happy kitten is an authenticated kitten!
